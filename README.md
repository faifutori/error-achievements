# エラー実績解除

コードを**壊すこと**が目的の、横スクロール2D探索ゲームです。
静かなダークテーマの世界を歩き、4つの端末に入ったPythonコードを
自由に書き換えて実行し、発生したエラーの種類に応じてバッジを集めます。

敵もタイマーもゲームオーバーもありません。
「まだ見ぬエラーを予想して、壊し方を考える」推理と収集のゲームです。

## 遊び方

| 操作 | 内容 |
|---|---|
| ← → | 移動 |
| ↑ | ジャンプ（長押しで高さが変わる） |
| ↓（端末の前で） | 選択：コードパネルを開く |
| Ctrl+Enter（パネル内） | コードを実行 |
| Esc（パネル内） | パネルを閉じる |
| Space または Tab | バッジコレクション画面の開閉 |

パネル内では「修復」ボタンでいつでも元の正しいコードに戻せます。
壊し方が思いつかないときは「ヒント」ボタンで、まだ取っていない
バッジの出し方のヒントが1つ表示されます。
エラーで解除されたバッジは、光の柱とともに空から降りてきます。
変更した行は左端の「▶」マーカーと背景色でハイライトされます。

## 起動方法

ビルド不要です。次のどちらかで動きます。

1. `index.html` をブラウザでダブルクリックして開く
2. VS Code の Live Server で開く（推奨。Pyodideの読み込みが安定します）

インターネット接続があれば **Pyodide（本物のPython）** で実行されます。
オフラインや読み込み失敗時は、変更→エラー対応表による
**簡易シミュレーション** に自動フォールバックし、
4種すべてのバッジが最後まで取得できます（画面下部に動作モードを表示）。

## GitHub Pages での公開

このリポジトリをそのまま Push し、
Settings → Pages → Branch: `main` / `(root)` を選ぶだけで公開できます。

## バッジ一覧（全10種）

| エラー | バッジ名 | レアリティ |
|---|---|---|
| SyntaxError | 構文崩壊者 | Common |
| IndentationError | インデントの反逆者 | Common |
| NameError | 幽霊変数使い | Common |
| TypeError | 型の反乱軍 | Common |
| IndexError | 境界突破者 | Uncommon |
| KeyError | 鍵なき扉の探索者 | Uncommon |
| ImportError | 輸入禁止令の違反者 | Uncommon |
| ValueError | 値の錬金術師 | Rare |
| AttributeError | 属性迷宮の探索者 | Rare |
| ZeroDivisionError | ゼロ除算の勇者 | Epic |

ModuleNotFoundError は ImportError の一種として扱われます。
上記10種以外のエラーはバッジにはなりませんが、
「未知のエラーを発見した」として実行ログに記録されます。

全バッジの出し方は同梱の「答え.txt」を参照してください（ネタバレ注意）。

## 動作確認済みの壊し方リスト

以下はすべて、Pyodide実行・フォールバック実行の両方でテスト済みです。
全10種の詳しい手順は「答え.txt」にあります。以下は代表例です。

### SyntaxError（構文崩壊者）
- 端末1で `print("name:", name)` の閉じ括弧 `)` を消す
- 端末2で `if hp > 50:` の末尾のコロン `:` を消す

### IndentationError（インデントの反逆者）
- 端末2で `print("元気です")` の行頭の字下げ（スペース4つ）を消す
- 任意の端末で、1行目の先頭に余計なスペースを入れる

### NameError（幽霊変数使い）
- 端末1で `print("name:", name)` を `print("name:", nam)` に書き換える
  （存在しない変数名を呼ぶ）

### TypeError（型の反乱軍）
- 端末3で `print(add(2, 3))` を `print(add(2, "3"))` に書き換える
- 端末1で `print("level: " + level)` という行を書く（文字列 + 数値）
- 端末4で `print(len(5))` と書く（数値に len() を使う）

### おまけ（バッジ対象外・ログに記録される）
- 任意の端末で `print(1 / 0)` → ZeroDivisionError
- 端末4で `print(items[9])` → IndexError

## 行動ログ

Space/Tab画面の「ログをJSONで保存」で、以下を含むJSONをダウンロードできます。

- 参加者ID（ブラウザごとに固定）・開始/終了日時・セッション時間
- 実行回数（合計・正常・エラー）
- キー入力回数（総数）と編集イベント数
- エラー種別ごとの発生回数、未知のエラー発見記録
- 取得バッジと種類数、スコア合計（Common=10 / Uncommon=20 / Rare=30 / Epic=50）
- 実行間隔の統計（最短・最長・平均・中央値、秒）と生データ
- 無操作時間の統計（1秒以上の間隔が対象。最短・最長・平均・中央値、秒）と生データ
- バッジ取得の記録（壊したコード・エラー文つき）と実行イベントの時系列

要約はコレクション画面にも表示されます。

「ログをCSVで保存」では、Excelでそのまま開ける2つのCSVがダウンロードされます。
- `log-summary-<ID>.csv` … 1参加者=1行のサマリー（回数・スコア・エラー種別ごとの回数・実行間隔と無操作時間の統計）。複数人のファイルを縦に結合して分析できます
- `log-events-<ID>.csv` … 1実行=1行の時系列（時刻・端末・正常/エラー・エラー種別・実行エンジン）

## アセット差し替え

すべてのスプライトはプレースホルダ（Canvas描画）で動きますが、
`assets/` フォルダに以下のファイル名でPNGを置くと自動で読み込まれます。
**スプライトシートがあれば優先、なければ個別ファイル、
どちらもなければプレースホルダ**の順です。

| ファイル名 | サイズ | 内容 |
|---|---|---|
| `player.png` | 128×48 | 32×48を4コマ横並び（1=待機、2=歩行1、3=歩行2、4=ジャンプ） |
| `player-1.png`〜`player-4.png` | 各32×48 | 上記の個別ファイル版 |
| `tile_ground.png` | 32×32 | 地面タイル |
| `tile_ground-1.png` / `tile_ground-2.png` | 各32×32 | 模様違い2種 |
| `tile_block.png` | 32×32 | 浮きブロック |
| `tile_block-1.png` / `tile_block-2.png` | 各32×32 | 模様違い2種 |
| `terminal.png` | 128×48 | 32×48を4コマ横並び（1=変数、2=条件分岐、3=関数、4=データ構造） |
| `terminal-1.png`〜`terminal-4.png` | 各32×48 | 上記の個別ファイル版 |
| `badge-1.png`〜`badge-10.png` | サイズ自由（64×64推奨） | ゲーム内では32×32に縮小表示。番号は上のバッジ一覧の順（1=SyntaxError〜10=ZeroDivisionError） |
| `bg_sky.png` | 640×360 | 横シームレス背景 |

## 構成

```
index.html   … エントリポイント
style.css    … UIスタイル（DotGothic16 / Noto Sans JP / monospace）
game.js      … ゲーム本体（描画・物理・バッジ・ログ）
python.js    … Python実行層（Pyodide + フォールバック）
levels.js    … ステージと端末コードの定義
assets/      … PNGを置くと差し替わる（無くても動く）
```

## サウンド

標準では、ファイル不要のレトロ電子音（Web Audioで合成）が鳴ります。
Space/Tabのコレクション画面に **BGM・効果音のON/OFFと音量** の設定があります
（設定はブラウザに保存されます）。

`assets/sounds/` に以下のファイル名でmp3を置くと、自動でそちらが優先されます。
[効果音ラボ](https://soundeffect-lab.info/) などの素材をリネームして置くだけでOKです。

| ファイル名 | 鳴る場面 | 探すときのキーワード例 |
|---|---|---|
| `bgm.mp3` | 常時ループ再生 | ※BGMは効果音ラボにはないので DOVA-SYNDROME や甘茶の音楽工房などで「環境音・アンビエント・静かめ」 |
| `jump.mp3` | ジャンプ | ジャンプ、ぴょん |
| `open.mp3` | 端末を開く | 決定、カーソル、ピッ |
| `ok.mp3` | 正常実行 | 決定ボタン、正解 |
| `error.mp3` | エラー発生 | ブザー、エラー、不正解 |
| `descend.mp3` | バッジ降臨 | キラキラ、天使、神秘 |
| `collect.mp3` | バッジ取得 | 回復、アイテム取得、レベルアップ |
| `clear.mp3` | 全バッジ達成 | ファンファーレ、クリア |
| `hint.mp3` | ヒント表示 | ひらめき、ポン |
